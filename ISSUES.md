## Open

### #1 — Optimize slider attacks with magic bitboards
**Type:** performance (not correctness) · **Area:** `Games/Chess/Core/Private/Bitboard.h`

Sliders (bishop / rook / queen) currently use straightforward **ray-scanning**
(`SlideRay`, `BishopAttacks`, `RookAttacks`): loop outward in each direction until a
blocker. This is correct — the full perft suite passes — but it is not the fastest
approach, and "the fastest chess app" is an explicit project goal.

Replace ray-scanning with **magic bitboards**: precomputed per-square attack tables
indexed by a perfect-hash (multiply + shift) of the relevant occupancy, turning each
slider attack into a table lookup. Standard technique for a top-speed generator.

**Acceptance criteria**
- Perft values unchanged — the existing `chess_rules_tests` suite must still pass
  bit-for-bit (correctness is the safety net for the optimization).
- Slider attack generation measurably faster; add a small perft nodes/sec benchmark.
- No third-party libraries — magic numbers + tables generated by our own code
  (a one-time generator is fine).

**Surfaced from:** CodeViewer session "Chess Move Generation", step *"Slider attacks
by ray-scan"* (the note flags ray-scan as the spot to optimize later).
